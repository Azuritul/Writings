# 記事執筆スタイルガイド

Zennなどに投稿する日本語技術記事を書く際のトーン・文体ガイドライン。

## トーン

- 経験を振り返って率直に語るスタイル。教える側ではなく、やってみた側の視点
- 正直に書く — 失敗、想定外の結果、判断ミスも隠さない
- 感想は断定的に書く（「一番コスパが良さそうだった」「今は選ばなくてもいいと思っている」）
- 適度にカジュアル。フレンドリーだけどふざけすぎない

## 文体

- 一人称は基本省略。必要なときは「自分」
- ですます調ベース。ただし短い感想や振り返りは「〜だった。」「〜だと思った。」で切る
- 短文のリズムを大事にする（「以上だけだった。」「問題はここから。」）
- 括弧書きで補足や本音を挟む（「（正直これは想定外だった）」「（コスト的にペイしないのではないか？）」）
- 長い文を避ける。一文一意を意識

## 構造

- データや比較は表で整理する
- コードブロック・設定例は実際に動くものを載せる
- 地の文はカジュアルに、データ部分は構造的に
- Issue例やログなど実物を見せて説得力を持たせる

## Zenn frontmatter

- topicsは最大5個まで
- topicsに記号やスペースは使えない（例: C++ → `cpp`、C# → `csharp`）

## 避けること

- AI感のある文章（「〜していきましょう」「〜について解説します」のような教科書調）
- 過度な丁寧語の連続（「〜させていただきます」「〜となっております」）
- 感情の誇張（「めちゃくちゃ感動した！」）
- 空虚な前置き（「昨今のAI技術の発展により〜」）
